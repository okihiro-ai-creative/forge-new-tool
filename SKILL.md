# Forge New Tool

このスキルは、ユーザーとの要件定義からPythonツールを生成し、構築・起動確認まで進めるための手順書です。

## 1. 要件定義（Phase 0: Deep-Dive Dialogue）

**最重要:** ユーザーがツールの要望を伝えてきた際、即座に構築を開始してはならない。

1. **深掘り対話**: 以下の観点を中心にユーザーと対話し、理想の形を明確にする。
   - **ターゲット**: 誰が、いつ、どこで使用するツールか。
   - **コア体験**: そのツールを使うことで解決したい最大の課題は何か。
   - **機能詳細**: 必要な入力、期待する出力、GUIに求める操作感。
2. **設計案の提示**: 対話に基づき、どのようなツールを作るか（構成・機能一覧）を要約して提示し、ユーザーの承認を得ること。

## 2. 準備 (Prerequisites)

- ユーザーの承認を得たら、プロジェクト内にあるツール生成用プロンプトまたは設計テンプレートを読み込む。
- テンプレートが存在しない場合は、ユーザーに確認してから、今回の要件に合わせた最小構成の `build.py` を設計する。

## 3. 格納ルール (Location Rules)

- **ルート基準**: このスキル実行時のカレントディレクトリをプロジェクトルートとする。
- 新規ツールは、プロジェクトルート直下のツール格納フォルダに作成する。
  - 推奨例: `./tools/{ToolName}` または `./apps/{ToolName}`
- 既存の設定フォルダ、スキル本体フォルダ、認証情報フォルダへの作成は禁止。

## 4. 生成プロセス (Build Process)

### Step 1: `build.py` の生成
読み込んだプロンプトまたは設計テンプレートの内容に従い、ユーザーの要望を満たす `build.py` を作成する。
- 生成ルールがある場合は必ず守る。
- ターゲットディレクトリが存在しない場合は作成する。

### Step 2: 自動実行 (Auto-Execution)
`build.py` を作成したら、生成対象・書き込み先・実行内容を確認してから実行する。

```bash
python build.py
```

### Step 3: 環境構築 (Setup)
展開されたディレクトリに移動し、プロジェクトに含まれるセットアップ手順を実行する。

Windowsの例:
```bat
call _setup.bat
```

macOS / Linuxの例:
```bash
bash ./setup.sh
```

### Step 4: 起動確認 (Verification)
デバッグ用コマンドを実行して、エラーなくGUIまたはCLIが起動することを確認する。
GUIが起動確認できたタイミング、またはログに `Listening` 等が表示されたら成功とみなす。

## 5. 完了報告
- 作成されたツールのパス
- セットアップ結果
- 起動確認の結果
これらを報告して終了する。
